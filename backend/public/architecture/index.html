TGK Wealth Demo

Built for one demo, structured for many.

TGK combines two demo portals, one reusable backend, and live DocuSign services. The reusable backend is the accelerator for future bespoke demo assets without turning the demo itself into a large application.

Audience-first

Open a portal, run a workflow, and show the result.

Reusable backend

Auth, proxying, persistence, and extension plumbing live in one place so future bespoke demos start faster.

Lightweight to host

Static frontends and a small backend keep the stack portable across local and cloud environments.

Architecture At A Glance

How the experience is organized

Each layer has one job: tell the story, support the story, or power the live workflow.

Experience Layer
Advisor and Investor Portals

Two lightweight portals show the same demo from advisor and investor perspectives.

Launcher Choose the story and enter the right portal.
Advisor View Client book, agreements, onboarding, and configuration controls.
Investor View Accounts, documents, tasks, and workflow follow-up.
Support Layer
Reusable TGK Backend

One backend handles shared plumbing and speeds up future bespoke demo builds.

Auth and Consent Handles DocuSign consent and token retrieval.
API Proxy Routes DocuSign API calls through one controlled path.
Demo Data Store Stores employees, customers, envelopes, and tasks in SQLite.
Workflow Layer
DocuSign IAM

Live DocuSign services provide the workflow moments that make the demo feel real.

Workflows Runs workflows such as account opening and asset transfer.
Extension App Lets workflows read and write to TGK database.
Audience Journey

What someone sees during the demo

The platform is built around a clean setup-to-outcome demo flow.

1

Open the story

Start from the launcher or go straight into a portal. Branding, mode, and workflow settings shape the story without a rebuild.

2

Trigger a live workflow

The advisor or investor experience launches a live Workflow Builder flow through the backend.

3

Show the outcome

The portals refresh so the audience can see documents, tasks, and customer state change.

Why It Is Built This Way

The design principles behind the demo

The goal is a credible, reusable demo foundation, not a production platform.

Simple frontends

Static portals keep iteration and hosting easy.

Reusable backend

Shared plumbing accelerates future bespoke demo assets.

Real workflow moments

Live IAM and Workflow Builder integration keeps the story grounded in real DocuSign capability.

Clear demo narrative

The architecture supports a presentation flow that is easy to explain to non-technical audiences.

In plain terms: the portals tell the story, the reusable backend keeps future demos fast to build, and DocuSign IAM plus Workflow Builder provide the live workflow moments.
