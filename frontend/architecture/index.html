Architecture

Built for one demo, structured for many.

TGK Wealth is the current financial services experience, but underneath it is a reusable demo platform. The frontend tells the wealth-management story. The backend handles the infrastructure that future vertical demos should not have to rebuild.

Current Use

One advisor portal and one investor portal show DocuSign IAM in a believable operating context for Solution Consultants.

Platform Intent

Future demos should be able to swap in a new static frontend, a new app slug, and new seeded data while reusing the same backend services.

System Overview

The demo starts in a static web portal, runs through reusable backend services, and then coordinates DocuSign IAM, Maestro writeback, third-party APIs, and the demo data store.

Static Frontend

Advisor Portal

Operator experience for households, documents, tasks, and guided workflow initiation.

Static Frontend

Investor Portal

Client experience for holdings, reviews, transfers, and document-driven self-service steps.

Reusable Platform

App-Scoped Demo Backend

One modular service owns auth, proxying, data APIs, Maestro integration, and webhook intake across demo experiences.

DocuSign Auth

Stores connection state by app slug and manages consent, session, and account selection.

Generic Records API

Reuses the same employee, customer, task, and envelope model across demo frontends.

Proxy Layer

Lets the frontend call external services through a consistent CORS-friendly backend path.

Maestro Bridge

Supports workflow-triggered writeback so launched Maestro experiences can update TGK records.

DocuSign

IAM + Maestro

Account opening and transfer workflows start here, then write state back into the demo via Maestro.

External Systems

APIs + Demo Store

Third-party data flows through the proxy, while SQLite keeps the demo dataset portable and easy to seed.

Static at the edge

The presentation layer stays lightweight and easy to host anywhere.

Reusable in the middle

The backend carries the durable integration and data machinery.

DocuSign-centered

IAM and Maestro stay central to the product story, not bolted on afterward.

Portable demo data

The dataset is seeded through public APIs and can be replaced without backend rewrites.

What Is Specific vs Reusable

The wealth-management story is intentionally separable from the underlying services. That is what makes the platform useful beyond TGK Wealth.
